# AWS Security Hub

## 1. Service/Topic Overview
- **Purpose**: Centralized security posture management service that aggregates, organizes, and prioritizes security findings from multiple AWS services and third-party products into a single pane of glass
- **Service Level**: Regional (with cross-Region aggregation capability)
- **Key Use Cases**:
  - Centralized security findings dashboard across multiple AWS accounts and Regions
  - Automated compliance checks against industry standards (CIS, PCI DSS, NIST, FSBP)
  - Aggregating findings from GuardDuty, Inspector, Macie, and third-party tools
  - Automated remediation workflows via EventBridge integration
  - Security posture scoring and trend analysis for AI/ML workloads
- **Exam Weight**: Medium
- **Exam Domain**: Domain 5 — Security, Compliance, and Governance for AI Solutions (14%)
- **Task Statements**:
  - Task 5.1: Identify AWS services and features for securing AI systems
  - Task 5.2: Recognize governance and compliance requirements for AI systems

## 3. Core Concepts

### Architectural Patterns
- **Hub-and-Spoke Model**: Security Hub acts as the central hub receiving findings from multiple spoke services (GuardDuty, Inspector, Macie, Config, third-party tools)
- **Multi-Account Architecture**: Delegated administrator account manages Security Hub for all organization member accounts
- **Cross-Region Aggregation**: Aggregation Region collects findings from linked Regions for unified view

### Key Terminology
| Term | Definition |
|------|-----------|
| **Finding** | Observable record of a security check or detection |
| **Exposure Finding** | Finding generated by correlating signals from multiple sources |
| **Signal** | A contributing finding from CSPM, Config, Inspector, or other services |
| **Trait** | Security deviation type: Assumability, Misconfiguration, Reachability, Sensitive Data, Vulnerability |
| **OCSF** | Open Cybersecurity Schema Framework — standard format for security findings |
| **ASFF** | AWS Security Finding Format — standardized finding format used by Security Hub CSPM |
| **Security Score** | Calculated from security check results to measure compliance posture |
| **Automation Rules** | Rules to automatically update or suppress findings based on criteria |

### Data Flow
```
AWS Services (GuardDuty, Inspector, Macie, Config)
        ↓
    Security Hub (Aggregation + Normalization via OCSF/ASFF)
        ↓
    Findings Dashboard + Security Scores
        ↓
    EventBridge → Automated Remediation / Ticketing (Jira, ServiceNow)
```

### Integration Points
| Service | Integration Type | Purpose |
|---------|-----------------|---------|
| **Amazon GuardDuty** | Sends findings → Security Hub | Threat detection findings |
| **Amazon Inspector** | Sends findings → Security Hub | Vulnerability assessment findings |
| **Amazon Macie** | Sends findings → Security Hub | Sensitive data discovery findings |
| **AWS Config** | Sends findings → Security Hub | Resource configuration compliance |
| **Amazon EventBridge** | Receives findings ← Security Hub | Automated remediation triggers |
| **AWS Organizations** | Management integration | Multi-account management |
| **AWS CloudTrail** | Logging | Audit Security Hub API calls |
| **Third-party tools** | Bidirectional | Jira Cloud, ServiceNow ITSM |

## 4. Key Features & Components

### Exam-Tested Features
- **Security Standards**: Automated checks against:
  - AWS Foundational Security Best Practices (FSBP)
  - CIS AWS Foundations Benchmark
  - PCI DSS (Payment Card Industry Data Security Standard)
  - NIST (National Institute of Standards and Technology)
- **Findings Aggregation**: Normalizes findings from 50+ AWS and third-party sources into OCSF format
- **Security Scores**: Quantitative measure of compliance posture per standard
- **Automation Rules**: Automatically modify, suppress, or escalate findings
- **Cross-Region Aggregation**: View findings from all Regions in a single aggregation Region
- **Insights**: Correlated collections of findings that identify security areas requiring attention
- **Exposure Findings**: Correlated findings showing broader security weaknesses

### Configuration Options
- Enable/disable specific security standards
- Configure automation rules for finding management
- Set up cross-Region aggregation
- Designate delegated administrator account
- Configure third-party integrations (Jira, ServiceNow)

### AI/ML Security Relevance
- Monitor security posture of SageMaker resources (endpoints, notebooks, training jobs)
- Track compliance of AI/ML infrastructure against security standards
- Aggregate findings related to data protection for ML training data in S3
- Monitor IAM configurations for AI service access
- Detect misconfigurations in AI/ML supporting infrastructure

## 5. Exam Focus Areas

### Common Question Types
- **Scenario**: "A company needs a centralized view of security findings across multiple AWS accounts running AI/ML workloads" → **Security Hub**
- **Scenario**: "An organization wants to automatically check compliance against CIS benchmarks" → **Security Hub with CIS standard enabled**
- **Scenario**: "A team needs to automate remediation when a security finding is detected" → **Security Hub + EventBridge**
- **Comparison**: "Which service aggregates findings vs. which generates findings?"

### Gotchas
- ⚠️ Security Hub does NOT generate threat detection findings — that's **GuardDuty**
- ⚠️ Security Hub does NOT scan for vulnerabilities — that's **Inspector**
- ⚠️ Security Hub does NOT discover sensitive data — that's **Macie**
- ⚠️ Security Hub does NOT retroactively process findings from before it was enabled
- ⚠️ Security Hub requires **AWS Config** to be enabled for security checks to work
- ⚠️ For full CIS compliance, Security Hub must be enabled in **all supported Regions**
- ⚠️ Security Hub is Regional — use cross-Region aggregation for unified view

### Comparison Points

| Feature | Security Hub | GuardDuty | Inspector | Macie |
|---------|-------------|-----------|-----------|-------|
| **Primary Function** | Aggregate & assess | Threat detection | Vulnerability scanning | Sensitive data discovery |
| **Generates Findings** | Compliance checks + exposure findings | Threat findings | Vulnerability findings | Data sensitivity findings |
| **Sends to Security Hub** | N/A (is the hub) | ✅ Yes | ✅ Yes | ✅ Yes |
| **Compliance Standards** | ✅ CIS, PCI, NIST, FSBP | ❌ | ❌ | ❌ |
| **Security Score** | ✅ Yes | ❌ | ❌ | ❌ |
| **Automation Rules** | ✅ Yes | ❌ | ❌ | ❌ |

### Decision Tree
```
Need centralized security findings view?
├── YES → AWS Security Hub
│   ├── Need compliance checks? → Enable security standards (CIS, PCI, NIST)
│   ├── Need automated remediation? → Security Hub + EventBridge
│   └── Need multi-account? → Security Hub + Organizations (delegated admin)
└── NO
    ├── Need threat detection? → Amazon GuardDuty
    ├── Need vulnerability scanning? → Amazon Inspector
    ├── Need sensitive data discovery? → Amazon Macie
    └── Need resource config tracking? → AWS Config
```

## 6. Best Practices

### Security
- Enable Security Hub in all Regions for comprehensive coverage
- Use delegated administrator account (not management account) for day-to-day operations
- Enable cross-Region aggregation for unified visibility
- Integrate with CloudTrail for audit logging of Security Hub API calls
- Use IAM least-privilege for Security Hub access

### Cost Optimization
- **Pricing Model**: Based on number of security checks and finding ingestion events
- **Free Trial**: 30-day free trial available
- Disable unused security standards to reduce check costs
- Use automation rules to suppress low-priority findings and reduce noise
- Consolidate findings using cross-Region aggregation instead of checking each Region separately

### Performance
- Use automation rules to reduce manual triage effort
- Configure EventBridge rules for real-time remediation
- Set up cross-Region aggregation to avoid switching between Regions
- Use insights to focus on highest-priority security areas

### Operations
- Monitor security scores regularly for compliance drift
- Review and update automation rules periodically
- Integrate with ticketing systems (Jira, ServiceNow) for finding management
- Use the dashboard for executive-level security posture reporting

## 7. Hands-On Labs

### Quick Setup
1. Navigate to AWS Security Hub in the console
2. Click "Go to Security Hub"
3. Enable desired security standards (FSBP recommended as starting point)
4. Review findings after initial assessment completes (~2 hours)

### CLI Commands
```bash
# Enable Security Hub
aws securityhub enable-security-hub --enable-default-standards

# Get findings
aws securityhub get-findings

# Get security score for a standard
aws securityhub get-security-control-definition --security-control-id EC2.1

# Enable a specific standard
aws securityhub batch-enable-standards \
  --standards-subscription-requests '[{"StandardsArn":"arn:aws:securityhub:::ruleset/cis-aws-foundations-benchmark/v/1.2.0"}]'

# Create automation rule to suppress low-severity findings
aws securityhub create-automation-rule \
  --rule-name "SuppressLowSeverity" \
  --rule-order 1 \
  --criteria '{"SeverityLabel":[{"Value":"LOW","Comparison":"EQUALS"}]}' \
  --actions '[{"Type":"FINDING_FIELDS_UPDATE","FindingFieldsUpdate":{"Workflow":{"Status":"SUPPRESSED"}}}]'
```

### Practical Exercise
1. Enable Security Hub with AWS Foundational Security Best Practices
2. Enable GuardDuty and Inspector to send findings to Security Hub
3. Create an automation rule to auto-archive informational findings
4. Set up an EventBridge rule to send critical findings to an SNS topic
5. Review the security score dashboard

## 8. Sample Questions

### Question 1
**A company runs AI/ML workloads across multiple AWS accounts and Regions. The security team needs a centralized dashboard to view all security findings and assess compliance against industry standards. Which AWS service should they use?**

A) Amazon GuardDuty  
B) AWS Config  
C) AWS Security Hub  
D) Amazon Inspector  

**Correct Answer: C**

**Explanation**: AWS Security Hub provides a centralized dashboard that aggregates security findings from multiple AWS services (GuardDuty, Inspector, Macie) across accounts and Regions. It also runs automated compliance checks against standards like CIS, PCI DSS, and NIST. GuardDuty (A) only detects threats, Config (B) tracks resource configurations, and Inspector (D) scans for vulnerabilities — none provide the centralized aggregation and compliance dashboard that Security Hub offers.

---

### Question 2
**A data science team wants to automatically remediate security findings when a misconfigured SageMaker notebook instance is detected. Which combination of services should they use?**

A) AWS Security Hub + Amazon SNS  
B) AWS Security Hub + Amazon EventBridge + AWS Lambda  
C) Amazon GuardDuty + AWS Lambda  
D) AWS Config + Amazon CloudWatch  

**Correct Answer: B**

**Explanation**: Security Hub detects the misconfiguration through its security standards checks and generates a finding. EventBridge receives the finding event and triggers a Lambda function to automatically remediate the issue. Option A would only notify, not remediate. Option C uses GuardDuty which focuses on threat detection, not configuration compliance. Option D could work for Config rules but doesn't provide the centralized finding management that Security Hub offers.

---

### Question 3
**Which of the following services sends findings TO AWS Security Hub? (Select TWO)**

A) Amazon EventBridge  
B) Amazon GuardDuty  
C) AWS CloudTrail  
D) Amazon Macie  
E) Amazon SNS  

**Correct Answer: B, D**

**Explanation**: Amazon GuardDuty (threat detection) and Amazon Macie (sensitive data discovery) both send their findings to Security Hub for centralized aggregation. EventBridge (A) receives findings FROM Security Hub for automation. CloudTrail (C) logs API calls but doesn't send findings to Security Hub. SNS (E) is a notification service, not a finding provider.

---

**Exam Tip**: Security Hub is the **aggregator**, not the **generator** of most findings. Remember the pattern: GuardDuty/Inspector/Macie **generate** findings → Security Hub **aggregates and assesses** → EventBridge **automates response**. For the AIF-C01 exam, focus on Security Hub's role in providing governance and compliance visibility for AI/ML workloads.
